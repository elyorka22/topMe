# Оценка ресурсов для города с населением 100,000

## Анализ нагрузки

### Предположения:
- Население города: **100,000 человек**
- Активные пользователи: **10-20%** (10,000-20,000 человек)
- Ежедневные активные пользователи (DAU): **5-10%** (5,000-10,000 человек)
- Среднее количество посещений на пользователя: **2-3 раза в день**
- Средний размер страницы: **500KB** (после сжатия)

### Расчет трафика:

#### Ежедневный трафик:
- Посещений в день: 10,000 × 2.5 = **25,000 посещений/день**
- Трафик в день: 25,000 × 500KB = **12.5 GB/день**

#### Месячный трафик:
- Посещений в месяц: 25,000 × 30 = **750,000 посещений/месяц**
- Трафик в месяц: 12.5 GB × 30 = **375 GB/месяц**

## Оптимизация для снижения трафика

### 1. Компрессия (автоматически на Vercel):
- Gzip/Brotli: экономия **60-70%** трафика
- Новый трафик: **375 GB → 112-150 GB/месяц**

### 2. Кеширование:
- Browser cache: экономия **40-50%** повторных запросов
- Новый трафик: **112-150 GB → 56-90 GB/месяц**

### 3. Lazy loading:
- Загрузка только необходимого кода
- Экономия **20-30%** начального трафика

### 4. CDN (автоматически на Vercel):
- Кеширование статики на edge серверах
- Снижение нагрузки на origin

**Итоговый трафик после оптимизации:** **~50-100 GB/месяц**
